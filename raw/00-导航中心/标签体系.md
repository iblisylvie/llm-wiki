# 标签体系

> 知识库标签使用规范

## 标签分类

### 1. 客户类型标签

用于标识目标客户群体

- `#客户类型/大厂` - 互联网大厂、头部科技公司
- `#客户类型/创业公司` - AI创业公司、初创企业
- `#客户类型/传统企业` - 传统行业的AI转型企业
- `#客户类型/云服务商` - 公有云、私有云服务提供商

### 2. 应用场景标签

用于标识具体应用场景

- `#场景/实时对话` - 实时对话AI、聊天机器人
- `#场景/代码生成` - AI编程助手、代码补全
- `#场景/智能客服` - 客服机器人、自动问答
- `#场景/内容生成` - 文本生成、创意写作
- `#场景/搜索推荐` - 智能搜索、个性化推荐
- `#场景/视频理解` - 视频分析、内容理解

### 3. 竞品标签

用于标识竞品相关内容

- `#竞品/Nvidia` - Nvidia相关（H20、H100等）
- `#竞品/Groq` - Groq推理芯片
- `#竞品/Google` - Google TPU
- `#竞品/其他` - 其他竞品

### 4. 更新频率标签

用于内容维护管理

- `#更新/每日` - 需要每日更新（如热点追踪）
- `#更新/每周` - 需要每周更新（如竞品动态）
- `#更新/按需` - 按需更新（如产品资料）

### 5. 内容状态标签

用于内容审核流程

- `#状态/草稿` - 草稿状态，未完成
- `#状态/审核中` - 待审核
- `#状态/已发布` - 已发布，可对外使用
- `#状态/已过期` - 内容已过期，需要更新或归档

## 使用规范

### 多标签组合

一个文档可以使用多个标签，建议组合方式：

```
客户类型 + 应用场景 + 内容状态
```

**示例**：
```markdown
---
tags:
  - 客户类型/大厂
  - 场景/实时对话
  - 状态/已发布
---
```

### 标签使用场景

#### 客户案例文档
```markdown
tags:
  - 客户类型/创业公司
  - 场景/代码生成
```

#### 竞品分析文档
```markdown
tags:
  - 竞品/Nvidia

```

#### 市场情报文档

```markdown
tags:
  - 客户类型/云服务商

```

## 标签查询

### Obsidian查询示例

**查找所有大厂客户相关内容**：
```
tag:#客户类型/大厂
```

**查找需要每日更新的内容**：
```
tag:#更新/每日
```

**查找草稿状态的文档**：
```
tag:#状态/草稿
```

**组合查询（大厂 + 实时对话场景）**：
```
tag:#客户类型/大厂 tag:#场景/实时对话
```

### Dataview查询示例

**列出所有待更新内容**：
```dataview
TABLE file.mtime as "最后更新"
FROM ""
WHERE contains(file.tags, "#更新/每日") OR contains(file.tags, "#更新/每周")
SORT file.mtime ASC
```

**按客户类型分组**：
```dataview
TABLE file.name as "文档"
FROM ""
WHERE contains(file.tags, "#客户类型")
GROUP BY file.tags
```

## 标签维护

### 新增标签流程

1. 确认新标签的必要性
2. 在本文档中添加标签定义
3. 更新相关文档的标签
4. 在主索引中更新标签列表

### 标签清理

定期（每季度）审查标签使用情况：
- 删除未使用的标签
- 合并相似标签
- 规范标签命名
