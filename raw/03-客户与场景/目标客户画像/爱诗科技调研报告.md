## 二、技术体系

### 2.1 核心技术路线

**完全自研，非基座微调。** 爱诗科技从创立之初即选择自研 DiT（Diffusion Transformer）架构，是国内首家落地 DiT 架构的视频生成创业公司，与 SD 系微调厂商模式有本质区别。`[官方]` `[量子位]`

### 2.2 算法层核心创新

#### 2.2.1 模型架构

- **自研时空注意力机制**：超越传统时空分离和 fullseq 架构，在空间和时间感知上显著提升 `[官方·V2技术博客]`
- **FullAttn + SparseAttn 混合使用**：高频层用稀疏注意力省计算，关键层用全量注意力保质量，在计算量与注意力精度之间动态平衡 `[官方·V5技术博客]`
- **自研专用 Tokenizer**：针对视频与图像生成专项训练，在高压缩比下保持重建质量与生成效果 `[官方·V5技术博客]`

#### 2.2.2 训练策略

- **RLHF（人类偏好拟合）**：通过人类偏好数据提升审美匹配度 `[官方·V5技术博客]`
- **极致蒸馏**：采用分布匹配损失优化模型采样轨迹，结合特征自约束损失实现自我监督，平衡生成速度与质量 `[官方·V5技术博客]`
- **加权 Flow 训练**：在传统 flow 模型基础上通过加权损失促进更快更优的收敛 `[官方·V2技术博客]`

#### 2.2.3 数据基础设施

- 自研视频内容理解标注系统，对数据进行多维度丰富标注 `[官方·技术博客]`
- 不同标签数据在训练阶段发挥不同作用，按训练策略分阶段使用 `[官方·技术博客]`
- 从海量数据中筛选精华，走精数据路线（类 DeepSeek 效率哲学）`[36氪]`

### 2.3 推理层核心技术

#### 2.3.1 算法侧推理优化

| 技术手段 | 原理 | 效果 |
|---------|------|------|
| IRE 瞬时响应引擎 | 时间轨迹折叠 + 引导校正 + 自适应稀疏采样，将采样步数从 50+ 压缩至 1–4 步 | 计算效率提升数百倍，实现实时生成 |
| 极致蒸馏 | 分布匹配损失 + 特征自约束损失 | V5 Fast 1080P 视频 30 秒内完成 |
| 混合 Attention | FullAttn/SparseAttn 动态切换 | 在不降速前提下扩大模型规模空间 |

#### 2.3.2 工程侧推理优化

依托字节 BytePlus 的 MLP（Machine Learning Platform）平台 `[品玩]`：

- **CPU/GPU 灵活配比**：推理使用全量 GPU，剩余 CPU 资源分配数据预处理，消除冗余
- **可抢占队列机制**：低优先级任务在算力充足时运行，高优先级实时推理任务到来时自动抢占 GPU 资源
- **效果**：推理集群整体 GPU 利用率提升 30%，训练集群利用率提升至 90% 以上 `[品玩]`
- **自动化运维**：低利用率自动关机、碎片自动整理、闲时任务调度、故障自愈全流程覆盖，支持单任务稳定运行数百小时不间断 `[品玩]`

#### 2.3.3 推理硬件

- 推理侧以 GPU 为主，CPU/内存辅助 `[品玩]`
- 具体GPU型号未公开披露；结合与字节 BytePlus 的深度合作，推测主要依托火山引擎 GPU 资源（H800/H20 等国内适配型号）`[推断]`
- 整体算力成本为海外同类公司的十分之一甚至更低 `[36氪·王长虎采访]`

#### 2.3.4 各版本推理性能

| 版本 | 分辨率 | 生成速度 |
|------|--------|---------|
| V3.5 Turbo | 720P | 最快 5 秒，平均 7–10 秒 |
| V4 | 360P | 最快 5 秒 |
| V5 Fast | 1080P | 30 秒以内 |
| V5 Fast | 360P 短视频 | 数秒内 |
| R1 | 1080P 实时流 | 接近实时，极低延迟 |

### 2.4 R1 的范式转变：从"视频生成工具"到"实时世界模型"

R1（2026年1月）是目前最激进的技术创新，由三大核心技术协同构成：

**① Omni 原生多模态基座**
文本、图像、视频、音频统一编码为连续 token 流，端到端训练于真实世界数据，在保证实时性的同时维持 1080P 分辨率并实现视听同步。

**② 自回归流式生成机制**
带记忆增强注意力的自回归机制，实现无限流式生成；早期帧持续指导后续帧，保证长时序物理一致性。与传统固定长度片段生成模式本质不同。

**③ IRE 瞬时响应引擎**
将传统扩散模型 50+ 步采样压缩至 1–4 步，为高并发服务与终端部署奠定基础。

> **技术争议** `[知乎分析]`：业界部分分析认为 R1 技术路线与 Meta Emu3.5 高度相似，可视为改良版 Emu3.5；"世界模型"定位更多是产品叙事，本质是"拥有超强短期记忆、能够连续演算的自回归生成系统"。（注：此为第三方独立观点，非爱诗科技官方表述）

## 三、产品与模型

### 3.1 模型版本全览

| 版本 | 发布时间 | 核心技术 | 关键亮点 |
|------|---------|---------|---------|
| V1 | 2023年10月 | Diffusion+Transformer | 业内首个可生成4K质量视频的AI视频模型，早于Sora `[官方]` |
| V2 | 2024年2月 | DiT 架构正式落地 | 国内首家落地 DiT 架构，实现"准实时"视频生成 `[官方]` |
| V3/V3.5 | 2024年10月 | 多模态特效，生成时间≤30秒 | "毒液变身"爆火，2个月新增全球用户超1000万 `[量子位]` |
| V4 | 2025年2月 | 5秒生成360P | 全球首家上线"人声+音效"同步 `[官方]` |
| V5 | 2025年8月 | RLHF+极致蒸馏+混合Attn | 图生视频全球 Top 2，文生视频 Top 3 `[Artificial Analysis]` |
| V5 Fast | 2025年11月 | 1分钟内生成1080P | 首创全视频关键帧追踪编辑（Modify功能）`[官方]` |
| V5.5 Omni | 2025年12月 | 分镜+音画同步 | 国内首个"成片级"叙事模型，一键直出分镜+音频 `[官方]` |
| V5.6 | 2026年初 | 混合扩散-Transformer | 视觉伪影减少40%，支持多关键帧控制（最多7帧链式串联）`[官方·技术博客]` |
| **R1** | **2026年1月** | **自回归流式+IRE实时引擎** | **全球首个通用实时世界模型，实时1080P交互式视频流** `[官方]` |

### 3.2 标志性功能创新

- **Character2Video**（2024年3月）：全球首个人物一致性模型，跨帧保持人物形象和背景高度一致
- **Video to Video**：基于实拍视频或影视片段进行延长生成
- **Modify 精修**：行业首个基于关键帧编辑的全视频推理与追踪，修改一帧自动同步全段
- **Agent 创作助手**（V5）：无需 prompt 技巧，一键生成专业级视频
- **多关键帧控制**（V5.6）：同时上传首帧和尾帧，AI生成中间所有帧，最多7帧链式串联

### 3.3 开源情况

**全部模型均未开源。** 走纯商业化路线，通过订阅制产品和开放API对外提供服务，与可灵、即梦同属闭源阵营。

## 七、风险与局限

### 7.1 技术层面

| 问题 | 现状 |
|------|------|
| 长序列误差累积 | 自回归生成中，微小预测误差逐步积累，影响长视频结构完整性 `[官方·R1技术报告]` |
| 物理精度与速度权衡 | 实时生成（IRE）在生成复杂度上有特定取舍，超写实物理仿真仍有差距 `[官方·R1技术报告]` |
| 角色一致性 | R1 在连续叙事和转场细节上仍有优化空间 `[第三方评测]` |
| 实时交互输入延迟 | AI生成速度远超用户手动输入 prompt 速度，语音交互仍待完善 `[第三方评测]` |

### 7.2 商业层面

- **全部闭源**：无开源社区生态，缺乏开发者自发传播效应 `[推断]`
- **高度依赖字节云基础设施**：算力和平台层面与 BytePlus 深度绑定，有一定供应链集中风险 `[品玩]` `[推断]`
- **订阅付费渗透率**：1亿用户中 MAU 仅 1600 万，付费转化率尚有提升空间 `[推断]`
- **竞争压力持续加剧**：谷歌 Veo 3 在文生视频评测位居全球第一，大厂资源优势明显 `[Artificial Analysis]`

### 7.3 合规层面

- 海外市场涉及数据隐私（GDPR 等）合规压力
- 国内视频生成内容监管政策持续演进，合规成本不可忽视
